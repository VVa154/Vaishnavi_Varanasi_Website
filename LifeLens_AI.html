Introduction

The increasing prevalence of stress, mental health concerns, and performance anxiety calls for accessible tools that provide personalized guidance. Artificial Intelligence (AI), particularly generative models, opens up avenues to address these needs. This project aimed to create an AI life coach that mimics the behaviour of a human coach while retaining user-specific knowledge over time. The AI system is designed to work offline, protect user data, and simulate human-like therapeutic conversations, offering a unique blend of technology and empathy.

Objectives

  • Develop a generative AI chatbot that interacts via text and audio
  • Embed long-term memory using ChromaDB and SQLite to store user interactions
  • Build a user-friendly frontend interface for conversations and habit tracking
  • Ensure offline functionality and data privacy
  • Integrate emotion detection and stop-word alerts for ethical responsiveness

Scope

  • Research Paper Search: via Semantic Scholar and arXiv APIs
  • Paper Summarization: using a locally served language model (Mistral via Ollama)
  • Paper Recommendation Engine: based on keyword relevance and embeddings
  • UB Faculty Research Matcher: matches user keywords with UB faculty research stored in a local JSON DB and ChromaDB

Technical Details

  • Backend-Ollama, ChromaDB, SQLiteFrontend-Streamlit
  • Interaction Flow-
  • User inputs(text/voice)
  • Voice-to-textconversion(via Google Speech-to-Text)
  • Memory retrieval(ChromaDB+SQLite)
  • LLM-generated empathetic response
  • UI displays reply+ stores conversation
  • SQLite- Structured database for storing user chats, emotional states, goals, and session history
  • ChromaDB-Vector database for storing and retrieving semantically meaningful conversation segments (e.g., therapy Q&A examples)
  • Voice Handling-Google Speech-to-Text API: Converts uploaded voice recordings into text. Originally attempted Bark (LLM for voice generation), but switched due to high RAM requirements (~20–25 GB)

Challenges

  • Bark LLM for voice too memory- heavy
  • Token limit optimization
  • Ethical responses for serious issues
